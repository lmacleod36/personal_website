Thesis

Keywords: Walkthroughs, Social Media, Video, On-boarding

Type:
Thesis
When:
August 2013 - August 2014

Description:
My thesis looks at how developers use video to share code through video. I was originally drawn to the on-boarding techniques used by open source projects, and this eventually lead me to video. This work also stems from my time on the CHISEL Tours project. Because of the breadth of public programming videos, my topic is focused on videos that walk the user through an existing code base.

My methodology has relied heavily on Charmaz's Grounded Theory work. A colleague and I have analyzed over 20 walkthrough videos, totalling over 8 hours of footage, with a high threshold for inter-rater reliability. Our analytical codes cover the characteristics and techniques developers use in these videos.

The second part of this thesis involves interviewing content creators to understand how and why developers create these videos. These interviews have allowed us to explore questions brought up from the video analysis phase.

Related Works:
CHISEL's Tours project, "How Developers Use and Create Video Walkthroughs" poster


Tags for IDA Pro

Keywords: Tagging, Software Development, GUI

Type:
DRDC Project
When:
May - August 2013
Tools:
Python, PySide Library, IDA Python, IDA Pro, IDA SDK, GitHub

Description:
Tags for IDA Pro is a Python based plugin for IDA Pro.

Tags for IDA Pro was developed for malware analysts at the Defence Research Department of Canada. It is based off of CHISEL's TagSEA project for Eclipse. Tags for IDA Pro gave analysts a way of tagging information in order to keep track of important locations and tasks. It also allowed them to share information with peers.

In Tags For IDA Pro the user can add tags via comments in IDA Pro. The Tags for IDA Pro plugin allows the user to filter, sort, and explore these tags using a tree and table view.

A previous student begun a prototype which I refined. Originally tag data was stored in external pickle files and used simple regular expressions. In rewriting the program I:

  • Stored tag data in the internal IDA Pro database
  • Re-wrote the event handling processes
  • Upgraded the regular expressions.
  • Added functionality for the user to add and manage custom tag categories
  • Implemented a process for modifing and importing tag data from external sources

Because of security constraints, GitHub was extremely useful to the client, giving them an outlet for input and logging issues during testing.

Related Works:
CHISEL's TagSEA project, "ICE: Binary Analysis That You Can See" paper.

Note: The final product is not publicly available but feel free to email me with questions about using PySide in IDA Pro. The initial setup of the libraries can be challenging to configure properly. (Keep in mind that it's been a while since I've worked with these tools!)


TagSEA

Keywords: Eclipse, Updating, Program Comprehesion, Tool Support

Type:
DRDC Project
When:
August 2012- February 2013
Tools:
Java, Eclipse, GitHub, Detective Work

Description:
CHISEL's TagSEA project was developed from 2006-2008 as a joint project with researchers from IBM. TagSEA was a "framework for tagging locations of interest within Eclipse. TagSEA combines ideas from social bookmarking (tagging) and geographic navigation (way-pointing) aiming to make finding information easier." There was no existing documentation for the project.

My task on this project was to update TagSEA so that it could work with modern versions of Eclipse, and eventually be integrated into other projects for the DRDC.

Related Works:
CHISEL's TagSEA SourceForge site, Tags for IDA Pro project, Tours project

Note: The updated version of TagSEA is not publicly available at the moment, it will be made public (with documentation!) in August 2014.


Reputation on Stack Overflow

Keywords: Data Mining, Community Detection, Statistics, Q & A sites

Type:
Class Work
When:
April - August 2013
Tools:
Python, SQLite, Pajek, R, Gephi

Description:
My term project for CSC 582S "Theoretical Models and Applications". The final paper will be published and presented at the 2014 International Symposium on Mining and Web.

In this project I mined the June 2013 Stack Exchange dump to explore user reputation scores on Q & A sites. I used community detection algorithms (specifically Blondel et al.) on over 35 GB of data, representing 12 randomly selected Stack Exchange sites. These algorithms determined the modularity of sub-communities on these sites.

From this data I created a number of weighted, directed graphs. These were analyzed with an exploratory method to formulate research questions. The research questions for this work were:

  • Is there a correlation between a user's reputation scoreand the number of unique tags they have commented on?
  • Do high reputation users belong to the same sub-communities?

The findings of this paper will be presented in late May 2014.


Tours

Keywords: Use Cases, Walkthroughs, Eclipse, Intergration

Type:
DRDC Project
When:
February - August 2013
Tools:
Java, Eclipse, Microsoft Expression Blend, GitHub

Description:
Tours is a sub-project within the TagSEA project. It was originally conceived to let programmers present their work in the Eclipse IDE. Think of is as Power Point for Eclipse. After I had made TagSEA compatable with the newest version of Eclipse, I began working with malware analysts at the DRDC to create tool requirements for an extended version of Tours. I created five detailed use cases for the features that would be added to the existing program. I then worked with two developers, who implemented these new features.
